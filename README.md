# Лабораторная работа: анализ динамических игр

Проект решает конечные динамические игры с совершенной информацией, заданные в виде ориентированного графа без циклов. Основной метод решения — обратная индукция. На выходе проект умеет строить текстовый отчёт, сохранять решение в JSON, визуализировать граф и запускаться через простой GUI.

## Что делает проект

- хранит игру как граф из вершин выбора и терминальных вершин;
- валидирует структуру входной модели;
- решает игру методом обратной индукции;
- сохраняет несколько оптимальных решений при равенстве;
- выводит дерево игры и пошаговый лог работы алгоритма;
- импортирует и экспортирует модели в JSON;
- строит граф с подсветкой оптимального пути;
- содержит встроенные примеры и генератор случайных игр;
- включает автотесты для ядра проекта.

## Ограничения модели

- поддерживаются только конечные игры с совершенной информацией;
- граф должен быть ацикличным;
- все вершины должны быть достижимы из корня;
- поддерживаются только вершины типов `decision` и `terminal`;
- в терминальной вершине размер `payoff` должен совпадать с числом игроков;
- в вершине выбора действия должны иметь уникальные названия;
- GUI не редактирует граф, а только загружает, генерирует, решает и сохраняет результаты.

## Среда

- проект проверялся на `Python 3.13.5`;
- для графического рендера используются `matplotlib` и `networkx`;
- тесты запускаются через `pytest`;
- если графические зависимости недоступны, экспорт графа падает в `.dot` вместо `.png`.

## Основные команды

Команды ниже предполагают, что Python-окружение уже настроено и зависимости доступны.

Показать встроенные примеры:

```bash
python main.py list-examples
```

Встроенные примеры:

- `bargaining`
- `centipede`
- `trust_model`

Выгрузить встроенный пример в JSON:

```bash
python main.py export-example bargaining examples_json/bargaining.json
python main.py export-example centipede examples_json/centipede.json
python main.py export-example trust_model examples_json/trust_model.json
```

Решить игру из JSON, вывести дерево, шаги алгоритма, сохранить картинку и JSON с решением:

```bash
python main.py solve examples_json/bargaining.json --print-tree --print-steps --render-png out/bargaining.png --save-solution-json out/bargaining_solution.json
```

Сгенерировать случайную игру и сохранить её:

```bash
python main.py random --players 3 --depth 3 --branching 2 --seed 42 --out examples_json/random_game.json
```

Запустить GUI:

```bash
python main.py gui
```

PowerShell:

```powershell
.\.venv\Scripts\python.exe main.py gui
```

## Параметры CLI

Команда `solve` поддерживает:

- `--print-tree` — добавить текстовое дерево игры в отчёт;
- `--print-steps` — добавить пошаговый лог обратной индукции;
- `--render-png PATH` — сохранить визуализацию графа;
- `--save-solution-json PATH` — сохранить найденные решения в JSON;
- `--epsilon VALUE` — считать близкие полезности равными при сравнении.

Команда `random` поддерживает:

- `--players` — число игроков, минимум `2`;
- `--depth` — глубина дерева, минимум `1`;
- `--branching` — ветвление, минимум `2`;
- `--seed` — seed генератора случайных чисел;
- `--title` — заголовок игры;
- `--out` — путь для сохранения JSON.

## GUI

Графический интерфейс построен на `tkinter` и закрывает демонстрационный сценарий без ручного редактирования графа.

В GUI можно:

- выбрать встроенный пример;
- открыть JSON-файл игры;
- сгенерировать случайную игру;
- решить игру и посмотреть текстовый отчёт;
- сохранить текущую игру в JSON;
- сохранить решение в JSON;
- сохранить граф или граф с подсветкой решения.

## Формат JSON

Модель игры задаётся объектом с полями `title`, `players`, `root` и `nodes`.

- `players` — список игроков с полями `id` и `name`;
- `root` — идентификатор корневой вершины;
- `nodes` — список вершин игры.

Для вершины выбора обязательны:

- `id`
- `kind: "decision"`
- `player`
- `edges`

Для терминальной вершины обязательны:

- `id`
- `kind: "terminal"`
- `payoff`

Поле `label` является необязательным для обоих типов вершин.

Пример:

```json
{
  "title": "Simple game",
  "players": [
    {"id": "P1", "name": "Игрок 1"},
    {"id": "P2", "name": "Игрок 2"}
  ],
  "root": "N0",
  "nodes": [
    {
      "id": "N0",
      "kind": "decision",
      "player": "P1",
      "label": "Старт",
      "edges": [
        {"action": "A", "to": "T1"},
        {"action": "B", "to": "T2"}
      ]
    },
    {
      "id": "T1",
      "kind": "terminal",
      "label": "Исход 1",
      "payoff": [3, 1]
    },
    {
      "id": "T2",
      "kind": "terminal",
      "label": "Исход 2",
      "payoff": [0, 2]
    }
  ]
}
```

## Что проверяет валидатор

- граф не пустой и содержит корневую вершину;
- идентификаторы игроков уникальны;
- все рёбра ведут в существующие вершины;
- в графе нет недостижимых вершин;
- в графе нет циклов;
- у вершины выбора указан существующий игрок и есть исходящие рёбра;
- у терминальной вершины нет исходящих рёбер и корректно задан `payoff`.

## Как формируется решение

Решение строится методом обратной индукции снизу вверх. Для каждой вершины выбора проект рассматривает оптимальные продолжения во всех дочерних подиграх, сравнивает полезность текущего игрока и оставляет все продолжения, дающие максимум. Если несколько ветвей равнозначны с учётом `epsilon`, они сохраняются как несколько оптимальных решений.

В текстовом отчёте можно получить:

- дерево игры;
- итоговые оптимальные решения и полезности;
- порядок сворачивания вершин;
- лог шагов обратной индукции.

## Архитектура проекта

- `main.py` — CLI и маршрутизация команд;
- `gui_main.py` — минимальная точка входа для GUI;
- `dynamic_games/models.py` — структуры `Game`, `Node`, `Edge`, `Player`;
- `dynamic_games/io_json.py` — чтение и запись игр и решений в JSON;
- `dynamic_games/validator.py` — проверка корректности графа;
- `dynamic_games/solver.py` — реализация обратной индукции;
- `dynamic_games/renderer_text.py` — текстовое дерево, отчёт и лог шагов;
- `dynamic_games/renderer_graph.py` — экспорт графа в PNG или DOT;
- `dynamic_games/examples.py` — встроенные модели из проекта;
- `dynamic_games/random_generator.py` — генератор случайных игр;
- `dynamic_games/workflows.py` — связка загрузки, валидации, решения и сборки отчёта;
- `dynamic_games/gui_app.py` — интерфейс на `tkinter`;
- `examples_json/` — примеры JSON-файлов;
- `tests/` — автотесты.

## Тесты

Основной набор тестов запускается так:

```bash
pytest -q
```

Тесты покрывают:

- JSON-ввод и JSON-вывод;
- валидацию некорректных графов;
- обратную индукцию на встроенных примерах;
- сохранение нескольких оптимальных решений;
- workflow для встроенных и случайных игр.
